# 需求分析培训摘要

本材料用于 RAG 检索，帮助 Agent 解释需求分析方法。流程控制规则以 Git Skill 为准。

## 核心目标

需求分析的目标是把客户的一句话想法，逐步澄清为可分析、可评估、可落地的需求。对 RPA 场景来说，重点不是一开始就问具体按钮和页面，而是先确认业务边界。

## 六个边界问题

1. 业务目标：机器人要帮助完成什么事情？
2. 触发条件：什么时候开始执行？
3. 完成条件：做到什么程度算结束？
4. 输入数据：机器人开始前需要哪些数据？
5. 操作系统：机器人需要在哪些系统、平台或文件中操作？
6. 输出结果：什么结果能证明任务完成？

## 分析顺序

先问边界，再问适配性，最后才进入流程细节。

边界不清时，不应直接设计流程。流程不清时，也不应急着生成最终需求文档。Agent 应先帮助客户补齐必要事实，并把不确定点显式列出来。

## 面向客户的表达方式

优先使用选择题，降低客户回答成本。每个选择题应保留“暂不确定”和“其他，请补充”的空间，让客户能表达非标准情况。
