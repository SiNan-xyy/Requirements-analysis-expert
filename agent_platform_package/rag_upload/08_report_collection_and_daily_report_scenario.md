# 报表采集与日报生成场景

本材料用于 RAG 检索，帮助 Agent 处理“从多个系统或文件采集数据，并生成日报、周报、月报、经营看板”的需求。电商日报是该类场景的一个样例，不应为每个客户场景单独建立 RAG。

## 适用需求

客户可能会这样描述：

- 自动统计多个电商平台的日数据，写入腾讯文档。
- 每天汇总广告平台消耗和转化，生成日报。
- 从多个系统导出销售数据，生成经营日报。
- 从库存系统读取数据，生成库存报表。
- 每天固定时间把运营数据写入在线表格。

## 典型流程

### 1. 加载报表配置

业务目的：

- 明确本次统计哪些数据源、日期、指标和目标报表。

可参考能力：

- 读取 Excel 或在线表格配置。
- 读取固定变量。
- 条件判断。

需要确认：

- 数据源清单是否固定。
- 报表日期口径。
- 目标报表模板。

### 2. 登录或打开数据源

业务目的：

- 进入平台后台、系统报表、文件目录或在线数据源。

可参考能力：

- 打开网页。
- 填写账号密码。
- 等待页面元素。
- 判断登录是否成功。

需要确认：

- 登录方式是否稳定。
- 是否存在验证码、扫码或设备确认。
- 账号权限是否覆盖所有数据源。

### 3. 进入报表或数据页面

业务目的：

- 找到包含目标指标的页面或文件。

可参考能力：

- 点击菜单。
- 输入查询条件。
- 等待报表加载。
- 判断页面包含目标字段。

需要确认：

- 报表入口是否固定。
- 指标字段位置是否稳定。
- 是否需要切换店铺、平台、账号或组织。

### 4. 选择日期和筛选条件

业务目的：

- 按业务口径获取对应日期或范围的数据。

可参考能力：

- 填写日期。
- 点击筛选。
- 条件判断。
- 等待结果刷新。

需要确认：

- 使用自然日、平台统计日、前一日还是自定义周期。
- 是否跨时区。
- 数据刷新时间是否晚于机器人执行时间。

### 5. 读取指标数据

业务目的：

- 获取 GMV、订单量、消耗、收入、退款、库存、转化等指标。

可参考能力：

- 获取网页元素文本。
- 获取表格数据。
- 循环相似元素。
- 读取下载报表。

需要确认：

- 指标名称和口径。
- 指标单位。
- 空值、0、无权限、未刷新如何区分。

### 6. 汇总和计算

业务目的：

- 按平台、店铺、部门、项目或日期汇总数据。

可参考能力：

- 条件判断。
- 变量运算。
- 统计分组求和。
- 数据表格处理。

需要确认：

- 汇总规则。
- 部分数据源失败时是否仍汇总成功部分。
- 是否需要标注失败来源。

### 7. 写入目标报表

业务目的：

- 把结果写入腾讯文档、在线表格、Excel 或系统报表。

可参考能力：

- 打开在线表格。
- 定位 Sheet。
- 写入指定行列或单元格。
- 写入后读取校验。

需要确认：

- 目标模板结构。
- Sheet 名称。
- 行列定位方式。
- 是否允许覆盖原值。

### 8. 记录日志和异常

业务目的：

- 保留执行证据，方便人工核查。

可参考能力：

- 写入日志。
- 生成异常清单。
- 截图。
- 保存文件。

需要确认：

- 日志字段。
- 异常清单保存位置。
- 是否需要截图。

### 9. 发送完成或异常通知

业务目的：

- 告知业务人员任务完成、失败或需要人工确认。

可参考能力：

- 飞书群通知。
- 钉钉群通知。
- 企业微信通知。
- 邮件通知。

需要确认：

- 通知渠道。
- 接收人。
- 通知触发条件。
- 通知内容。

## 常见分支

| 分支 | 可建议处理方式 | 是否必须客户确认 |
|---|---|---|
| 单个数据源登录失败 | 记录失败并继续其他数据源 | 是，需确认是否允许继续 |
| 报表数据为空 | 标记待确认，不自动填 0 | 是，需确认空值含义 |
| 指标字段找不到 | 记录异常并通知人工 | 否，可作为默认建议 |
| 部分数据源失败 | 只汇总成功部分并标注失败项 | 是，需确认业务是否接受 |
| 写入目标报表失败 | 保留源数据和日志，通知人工 | 否，可作为默认建议 |
| 源值与写入值不一致 | 进入人工复核 | 是，需确认是否要求自动核对 |

## 电商日报样例

示例需求：

“每天固定时间从淘宝/天猫、京东、拼多多等平台读取 GMV、订单量、退款等指标，按平台和店铺汇总后写入腾讯文档日报。”

应确认：

- 平台和店铺清单是否固定。
- 每个平台指标名称和口径是否一致。
- 是否包含退款、售后、取消订单。
- 报表日期口径。
- 腾讯文档写入位置。
- 部分平台失败时是否仍生成日报。

报告中应避免：

- 在未确认订单量时，把订单量写成已确认采集指标。
- 在未确认日期口径时，直接写“当日数据”。
- 在未确认腾讯文档登录方式时，写成“可无人值守写入”。

## 报告表达建议

对客版：

“当前需求属于报表采集与日报生成场景，具备较强自动化潜力。进入搭建前，需要确认数据源、日期口径、指标口径、目标报表位置和失败处理策略。”

实施对齐版：

“建议按配置加载、数据源登录、报表定位、指标读取、汇总计算、目标表写入、结果校验、异常通知拆分流程。每一步应展示可参考影刀能力、客户已确认信息和待确认项。”
