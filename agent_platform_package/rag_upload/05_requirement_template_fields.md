# 需求文档字段说明

本材料用于 RAG 检索，帮助 Agent 把澄清结果映射到需求文档字段。最终文档生成属于后续模块。

## 需求背景

说明为什么需要这个自动化，当前人工处理有什么问题，期望改善什么。

## 业务目标

说明机器人要完成的核心业务事项。对应基础澄清中的 `business_goal`。

## 起点和终点

起点说明流程什么时候开始，对应 `trigger`。

终点说明流程什么时候结束，对应 `completion_condition`。

## 输入数据

说明机器人开始前需要哪些数据，包括数据来源、字段、格式和稳定性。对应 `input_data`。

## 应用系统

说明机器人需要操作哪些系统、平台、文件或消息工具。对应 `operated_systems`。

## 输出结果

说明流程完成后产生什么结果，以及如何判断成功或失败。对应 `output_result`。

## 规则和判断

说明哪些判断可以写成明确规则，哪些仍依赖人工判断。对应 RPA 预筛中的 `rule_clarity`。

## 风险和前置条件

说明是否存在命名不统一、输入不稳定、页面不稳定、验证码、结果不可验证等问题。

当存在前置治理问题时，应先形成缺口或阻塞建议，而不是直接生成可开发需求。
