# 影刀常用能力中文卡片

本材料用于 RAG 检索，帮助 Agent 在流程拆解、异常设计和报告打包时，用业务专员能理解的方式说明“可参考的影刀能力”。本材料只提供能力方向，不提供精确点击路径、选择器、等待时间或指令参数。

## 使用原则

- 对客版报告只展示业务动作，不展示复杂指令名。
- 实施对齐版可以展示“可参考的影刀能力/指令方向”，但不要写成最终搭建步骤。
- 指令存在只是能力证据，不能单独代表需求一定可落地。
- 使用任何能力前，都要结合输入稳定性、规则清晰度、平台可操作性和结果可验证性判断。

## 能力卡 1：打开网页或进入系统

适用场景：

- 打开电商后台、ERP、CRM、腾讯文档、飞书表格等网页系统。
- 根据网址进入固定页面。

可参考能力：

- 打开网页。
- 获取已打开的网页对象。
- 恢复或激活浏览器窗口。

需要客户确认：

- 目标网址是否固定。
- 浏览器、账号、网络环境是否稳定。
- 是否需要登录、扫码、验证码或设备确认。

常见风险：

- 登录态失效。
- 页面地址变化。
- 浏览器版本或缩放比例影响元素识别。

报告表达建议：

“机器人先打开目标网页，并保留网页对象，供后续点击、读取或写入使用。”

## 能力卡 2：等待页面或元素加载

适用场景：

- 页面跳转后等待按钮、表格、结果区域出现。
- 报表刷新后再读取数据。

可参考能力：

- 等待元素出现。
- 判断元素是否可见。
- 判断网页是否包含指定文本或元素。

需要客户确认：

- 成功加载的页面标志是什么。
- 是否有固定提示文字、状态、按钮或表格区域。

常见风险：

- 页面加载慢。
- 元素偶发不可见。
- 弹窗遮挡目标区域。

报告表达建议：

“关键页面加载后，机器人先判断目标区域是否出现，再执行读取或写入，避免页面未加载完成导致误操作。”

## 能力卡 3：点击网页元素

适用场景：

- 点击查询、下载、提交、保存、筛选、切换 Sheet 或菜单。

可参考能力：

- 点击元素。
- 判断元素可见后点击。
- 捕获或复用网页元素。

需要客户确认：

- 按钮或菜单是否固定。
- 是否存在二次确认弹窗。
- 点击后成功标志是什么。

常见风险：

- 按钮位置变化。
- 点击后无响应。
- 点击触发高风险提交动作。

报告表达建议：

“机器人可模拟人工点击固定按钮，但高风险提交动作需要确认是否允许自动执行。”

## 能力卡 4：填写网页输入框

适用场景：

- 输入账号密码、订单号、日期、关键词、金额、备注等。

可参考能力：

- 填写输入框。
- 填写密码框。
- 清空后填写。

需要客户确认：

- 输入字段是否固定。
- 输入格式是否有要求。
- 是否需要先清空原内容。

常见风险：

- 输入框定位失败。
- 日期格式不一致。
- 字段校验失败。

报告表达建议：

“机器人可把已确认的数据填入固定输入框，填写前应先确认字段格式和页面校验规则。”

## 能力卡 5：读取网页文本或表格数据

适用场景：

- 从网页报表读取 GMV、订单量、状态、金额、库存、物流状态等。
- 从列表或表格中读取多行数据。

可参考能力：

- 获取元素文本。
- 获取表格数据。
- 循环相似元素。
- 判断网页包含指定文本。

需要客户确认：

- 读取字段在页面上的位置是否稳定。
- 指标名称、统计口径和单位是否固定。
- 数据为空时代表真实为 0，还是页面未加载或无权限。

常见风险：

- 指标位置变化。
- 表格分页。
- 字段名称相似但口径不同。
- 空值被误认为 0。

报告表达建议：

“机器人可以读取网页中的指标或表格数据，但需要先确认字段位置、统计口径和空值处理规则。”

## 能力卡 6：读取 Excel 或在线表格

适用场景：

- 读取客户清单、配置表、字段映射表、日报模板、异常清单。
- 按行读取待处理数据。

可参考能力：

- 打开或新建 Excel。
- 读取表格数据。
- 获取表格数据。
- 循环 Excel 内容。

需要客户确认：

- 文件路径或在线表格入口是否固定。
- Sheet 名称、表头位置、字段名称是否固定。
- 是否存在合并单元格、隐藏列、空行或格式变化。

常见风险：

- 文件被占用。
- 表头变化。
- 字段缺失。
- 数据格式不一致。

报告表达建议：

“机器人可以从固定表格读取配置或待处理数据，读取前需要确认表头、字段和数据范围。”

## 能力卡 7：写入 Excel 或在线表格

适用场景：

- 写入处理结果、日报数据、异常清单、执行日志。
- 将结果回写到指定行、列或单元格。

可参考能力：

- 写入内容至 Excel 工作表。
- 写入内容至数据表格。
- 保存或另存 Excel。
- 在线表格写入相关能力。

需要客户确认：

- 写入位置是否固定。
- 是否按行追加、按日期定位，还是写入指定单元格。
- 是否需要写入前备份。
- 是否允许覆盖原有内容。

常见风险：

- 写入位置错误。
- 模板结构变化。
- 权限不足。
- 写入后未保存。

报告表达建议：

“机器人可以把结果写入指定表格位置，但写入规则、覆盖规则和校验方式必须提前确认。”

## 能力卡 8：循环处理多条数据或多个对象

适用场景：

- 多平台、多店铺、多订单、多邮件、多文件逐个处理。

可参考能力：

- ForEach 列表循环。
- ForEach 字典循环。
- For 次数循环。
- 循环相似元素。

需要客户确认：

- 循环对象清单是否固定。
- 失败时是否继续处理后续对象。
- 是否需要记录每个对象的处理状态。

常见风险：

- 循环没有退出条件。
- 单个对象失败影响整体流程。
- 处理状态无法追踪。

报告表达建议：

“机器人可以按清单逐个处理对象，建议为每个对象记录成功、失败、跳过或待人工确认状态。”

## 能力卡 9：条件判断和分支处理

适用场景：

- 判断是否继续、跳过、重试、人工确认或终止。
- 根据状态、金额、分类、字段是否为空进行分支。

可参考能力：

- IF 条件。
- IF 多条件。
- Else IF。
- 判断元素可见、网页包含、文件存在。

需要客户确认：

- 判断条件是否能写清楚。
- 每个分支的处理动作是什么。
- 无法判断时交给谁处理。

常见风险：

- 业务规则含糊。
- 多条件冲突。
- 人工经验无法规则化。

报告表达建议：

“机器人可以执行明确的条件判断，但不能替业务方临场决定规则。规则不清时应先由业务方确认。”

## 能力卡 10：发送通知

适用场景：

- 流程完成通知。
- 异常通知。
- 人工确认提醒。

可参考能力：

- 飞书群通知。
- 钉钉群通知。
- 企业微信通知。
- 邮件通知。

需要客户确认：

- 通知渠道。
- 接收人或群。
- 通知内容字段。
- 哪些异常需要立即通知，哪些只进入日志。

常见风险：

- 机器人无发送权限。
- 群机器人配置缺失。
- 通知过多影响业务人员。

报告表达建议：

“机器人可以在完成或异常时发送通知，但通知渠道、接收人和触发条件需要客户确认。”

## 能力卡 11：记录日志和异常清单

适用场景：

- 记录处理时间、输入、输出、状态、失败原因、截图路径。
- 生成异常清单供人工复核。

可参考能力：

- 写入 Excel。
- 写入数据表格。
- 打印日志。
- 保存截图。

需要客户确认：

- 日志字段。
- 保存位置。
- 保留周期。
- 是否需要给人工复核清单。

常见风险：

- 日志字段不足，无法追溯问题。
- 异常只通知但没有记录。

报告表达建议：

“每次执行应保留日志和异常清单，便于业务人员核查和后续优化规则。”

## 能力卡 12：截图和证据保留

适用场景：

- 页面失败、写入失败、数据不一致、需要人工复核。

可参考能力：

- 截图。
- 保存文件。
- 记录截图路径。

需要客户确认：

- 哪些场景需要截图。
- 截图保存位置。
- 截图是否涉及敏感信息。

常见风险：

- 没有证据导致异常无法排查。
- 截图包含敏感数据，需要权限控制。

报告表达建议：

“关键异常建议保留截图或原始证据，方便人工复核和后续排查。”
