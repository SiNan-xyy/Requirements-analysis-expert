# 物流拦截案例

本材料用于 RAG 检索，帮助 Agent 理解“物流拦截”类需求如何从模糊变清晰。

## 初始需求

客户可能会说：“我想让机器人自动做物流拦截。”

这句话还不能直接进入开发，因为缺少触发、输入、系统、输出等边界信息。

## 澄清后的边界事实

业务目标：

- 自动完成物流拦截。

触发条件：

- 收到飞书机器人消息中的物流单号后开始。

输入数据：

- 物流单号。

操作系统：

- 飞书。
- 影刀商户后台。

完成条件：

- 拦截结果已发送回飞书。

输出结果：

- 飞书通知拦截成功或失败。

## RPA 预筛观察

这个需求相对适合进入后续 RPA 能力边界评估，因为：

- 输入数据明确。
- 动作具有重复性。
- 操作平台相对明确。
- 结果可以通过成功或失败通知验证。

仍需确认的风险包括：

- 商户后台是否存在频繁验证码。
- 页面入口是否稳定。
- 拦截失败原因是否需要分类处理。

这些问题属于后续能力边界评估和流程拆解的输入，不应在基础澄清阶段过度追问点击细节。
