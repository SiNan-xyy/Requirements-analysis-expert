# 需求记忆体模块

本模块是需求分析 Agent 的全局事实账本，用于把客户回答、模块判断、RAG 建议和待确认缺口分层记录下来。

## 目标

- 让模块流转依赖稳定的事实、缺口和门禁状态，而不是依赖模型一次性输出完美 JSON。
- 区分“客户已确认事实”和“Agent 推断建议”，避免把未确认内容当作开发依据。
- 支持跨模块继承：模块 2 到模块 6 都必须读取并更新同一份需求记忆体。

## 记录对象

- `Fxxx`：已确认事实，可以作为后续分析依据。
- `Ixxx`：推断建议，必须待客户确认，不能直接用于开发。
- `Gxxx`：待确认缺口，需要标记阻塞阶段。
- `Cxxx`：冲突或更正，用于记录客户推翻或修正既有信息。
- `Rxxx`：已废弃问题，用于避免重复提问。
- `Dxxx`：模块判断，用于记录阶段性结论。

## 使用原则

1. 每轮对话开始先读取需求记忆体。
2. 吸收客户回答后先更新记忆体，再决定是否进入下一模块。
3. 只有客户确认或上游模块确认的信息才能进入 `confirmed_facts`。
4. RAG 和 Agent 自行推断只能进入 `inferred_items` 或 `gaps`。
5. 如果事实不足但不阻塞当前模块，可以使用 `partial_ready` 携带缺口进入下一阶段。
